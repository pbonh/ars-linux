## Post-implementation addendum

After the initial build, the entrypoint was simplified. Snippets in this plan that show `sudo ansible-pull ... -i inventory/hosts ...` and playbooks with `hosts: all` are **stale**. Refer to the [design spec](../specs/2026-04-27-ars-linux-design.md) and the live repo files for current state.

Summary of changes:

- **No inventory file.** `inventory/hosts` was deleted. Both playbooks target `hosts: localhost` with `connection: local`. Per-host vars are loaded by hostname via a `pre_tasks` `include_vars` block (stat `host_vars/{{ ansible_hostname }}.yml`, include if present). This eliminates the `--limit` / hostname-to-inventory-key matching that ansible-pull's defaults imposed.
- **No `sudo ansible-pull`.** The system playbook is run as a normal user with `-K`; per-task `become` handles escalation. `user.yml` runs as the user with no escalation.
- **Two-play split in `system.yml`.** Play 1 gathers facts unprivileged; play 2 runs roles under `become: true`. Avoids slow-first-sudo races during fact gathering.
- **`ansible.cfg`:** dropped `inventory =` and `[inventory]` sections; switched `stdout_callback = yaml` to `stdout_callback = default` + `result_format = yaml` (the deprecated `community.general.yaml` callback); added `timeout = 60` and explicit `become_method = sudo`.
- **Justfile / README:** all `ansible-pull` commands updated; README's "Inventory" section replaced with "Per-host configuration".

Current one-liner on a bootc host:

```bash
ansible-pull -U https://github.com/pbonh/ars-linux.git -K system.yml
ansible-pull -U https://github.com/pbonh/ars-linux.git    user.yml
```

The step-by-step records below remain as written for historical accuracy.

---

**Goal:** Build an Ansible-driven layer (`ars-linux`) that customizes an unmodified upstream Zirconium bootc image by rendering a local Containerfile, building it with podman, and atomically activating it via `bootc switch`, plus a user-context playbook for distrobox/dotfiles.

**Architecture:** A single Ansible repository containing two playbooks (`system.yml` run as root, `user.yml` run as the desktop user) and a set of focused roles. Roles either (a) drop files into the live host's `/etc` (mutable) or (b) contribute to a Containerfile staged at `/var/lib/ars-linux/build/` that podman builds and `bootc switch` activates. Every host runs `ansible-pull` against this repo locally; per-host variation lives in `host_vars/<hostname>.yml`.

**Tech Stack:** Ansible 2.16+, Jinja2 templates, podman 5+, bootc 1.1+, Molecule (docker driver) for role idempotency tests, pytest + jinja2 for unit-level template tests, yamllint + ansible-lint for static checks, GitHub Actions for CI, Just for the user-facing CLI shortcuts.

**Spec:** [`docs/superpowers/specs/2026-04-27-ars-linux-design.md`](../specs/2026-04-27-ars-linux-design.md)

**Decomposition rationale.** Each role owns exactly one concern (one repo file family, one preset, one container family). The `bootc_image` role is the only orchestrator: it imports vars from the vars-only roles (`rpm_packages`, `third_party_rpms`, `appimages`) plus staged outputs from `dnf_repos` and renders the Containerfile. This split lets vars-only roles stay declarative and lets `bootc_image` be the single mutator of `/var/lib/ars-linux/build/`.

from __future__ import annotations
from pathlib import Path
import pytest
from jinja2 import Environment, FileSystemLoader, StrictUndefined

REPO_ROOT = Path(__file__).resolve().parent.parent


@pytest.fixture
def render_template():
    def _render(template_relpath: str, **vars_) -> str:
        full = REPO_ROOT / template_relpath
        env = Environment(
            loader=FileSystemLoader(str(full.parent)),
            undefined=StrictUndefined,
            keep_trailing_newline=True,
            trim_blocks=False,
            lstrip_blocks=False,
        )
        return env.get_template(full.name).render(**vars_)
    return _render

- [ ] **Step 3: Write `roles/bootc_image/files/relocate-opt.sh`**

```bash
#!/usr/bin/env bash
# Move /opt/{brave.com,vivaldi,cursor} to /usr/lib and rewrite desktop entries.
# bootc images cannot keep app payloads under /opt (it's mutable at runtime),
# so anything that ships there gets relocated at build time.
#
# ARS_RELOCATE_ROOT lets the unit test stage a fake filesystem; production
# leaves it unset so the script operates on the real / .
set -euo pipefail

ROOT="${ARS_RELOCATE_ROOT:-}"
APPS=("brave.com" "vivaldi" "cursor")

for app in "${APPS[@]}"; do
    src="${ROOT}/opt/${app}"
    dst="${ROOT}/usr/lib/${app}"
    [ -d "$src" ] || continue
    mkdir -p "${ROOT}/usr/lib"
    mv "$src" "$dst"

    # Rewrite Exec= lines in any .desktop files that point at the old path.
    if [ -d "${ROOT}/usr/share/applications" ]; then
        find "${ROOT}/usr/share/applications" -maxdepth 1 -name '*.desktop' -print0 |
            xargs -0 -r sed -i "s|/opt/${app}|/usr/lib/${app}|g"
    fi
done
```

## Out of scope (per the spec)

These were explicitly excluded by the design and intentionally not part of this plan:
- Publishing the derived image to a public registry.
- Automated unattended updates / scheduled rebuilds.
- Pinning specific upstream Zirconium digests.
- ISO build (`iso.toml`, `iso-nvidia.toml`).
- New cosign signing infrastructure.
- Reusability for other bootc distributions.

## Notes for the executing engineer

- **Cosign key placeholder.** [Task 5](#task-5-containers_policy-role) ships a placeholder PEM. Before the first real `bootc switch`, replace it with the upstream Zirconium cosign public key (from `mkosi.keys/cosign.pub` in the upstream repo). The CI Containerfile dry-run will warn but pass; the bootc switch on a real host will fail loudly until the key is real.
- **Pinned shas.** [Task 10](#task-10-third_party_rpms-role-vars-only), [Task 11](#task-11-appimages-role-vars-only), and [Task 16](#task-16-distrobox-role-user-side) all carry `"TO-BE-PINNED-AT-IMPLEMENTATION"`. Resolve each with the `just bump-*` helpers from [Task 18](#task-18-justfile-bump-helpers) before the first real `sync`.
- **Order of operations on a fresh host.**
  1. Boot a Zirconium ISO.
  2. `sudo dnf install -y ansible-core git just`.
  3. `git clone https://github.com/pbonh/ars-linux.git && cd ars-linux`.
  4. `just sync` → reboot when the marker file appears at `/var/lib/ars-linux/reboot-required`.
  5. Log in, then `just sync-user`.
- **Idempotency expectation.** The second consecutive `just sync` against the same git SHA must report zero changes from `bootc_image` (fingerprint matches → build skipped → no switch). If you see drift here, the fingerprint computation or build-context staging is non-deterministic; that's a bug, not config drift.
